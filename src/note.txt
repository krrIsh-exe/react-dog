// 🎮 What is OrbitControls?
// 👉 It lets you rotate, zoom, and move around your 3D object with mouse.

// Without it → camera is fixed.

// With it → you can explore the scene.

// 📦 Where it comes from?
// From:





// npm i @react-three/drei



// You import it like:





// import { OrbitControls } from "@react-three/drei"



// ✅ Basic Usage (Most Important)
// Inside <Canvas>:





// <Canvas>

// <mesh>

// <boxGeometry />

// <meshBasicMaterial color="orange" />

// </mesh>



// <OrbitControls />

// </Canvas>



// That’s it.

// Now:

// Left mouse → rotate
// Scroll → zoom
// Right click → pan
// 🧠 What OrbitControls Actually Does
// It controls:

// 👉 the camera

// Not the object.

// You are moving around the object,

// not moving the object.

// Very important concept.

// ⚙️ Important Props (Only the useful ones)
// 1️⃣ Disable zoom




// <OrbitControls enableZoom={false} />



// 2️⃣ Disable rotation




// <OrbitControls enableRotate={false} />



// 3️⃣ Limit vertical rotation




// <OrbitControls

// minPolarAngle={Math.PI / 4}

// maxPolarAngle={Math.PI / 2}

// />



// Polar angle = up-down rotation.

// 4️⃣ Smooth movement (very useful)




// <OrbitControls enableDamping dampingFactor={0.05} />



// Makes camera feel smooth and professional.

// 🧠 Professional Clean Version (Best Practice)




// <OrbitControls

// enableDamping

// dampingFactor={0.05}

// zoomSpeed={0.8}

// />



// This is what most devs use.

// ⚠️ Important Rule
// OrbitControls must be inside <Canvas>.

// Correct:





// <Canvas>

// <Scene />

// <OrbitControls />

// </Canvas>



// Wrong:





// <OrbitControls /> ❌ outside Canvas



// 🔥 Why Drei Version is Better
// Instead of manually doing:

// Drei version:

// Cleaner
// Less code
// React-friendly
// 🧩 Mental Model (Very Important)
// Think like this:

// Camera = Your Eyes

// OrbitControls = Your Neck

// You are turning your neck around the object.
